## Class 2: Repeat Loop Block

<!-- type: lesson -->

Class 2: Repeat Loop Block

![jump](https://dp4rp4tcpuoo1.cloudfront.net/R600V2/jump.png)

The *repeat* command in coding is like a shortcut that tells the computer to perform a specific action multiple times without needing to write each action individually. Imagine you want your character to jump 10 times in a row. Instead of typing the "jump" command 10 separate times, you can use a repeat command to tell the computer, "repeat the jump 10 times."

Here’s how it works: the repeat command creates a loop, where everything inside it will run once, then loop back to the beginning and run again until the specified number of times is reached. It’s efficient and keeps your code clean and short, especially when you need something done over and over!

Below is an example of how to make your robot drive in a square. 


![jump](https://dp4rp4tcpuoo1.cloudfront.net/R400V2/squarecode.png)

Here's an example below of how to make a robot drive in a square—first without a loop, then with a loop. Notice how the version with a loop is much cleaner and more efficient! 

![code](https://dp4rp4tcpuoo1.cloudfront.net/R400V2/sqw.png)

Without a loop, you have to repeat the same movement commands multiple times, making your code longer and harder to change later. With a loop, you can write the instructions once and let the robot repeat them, making your code more compact, easier to read, and simpler to modify if you want to adjust the square’s size or shape.

## Class 2: Forever Loop Block

<!-- type: lesson -->

Class 2: Forever Loop Block

![roller coaster](https://dp4rp4tcpuoo1.cloudfront.net/R400V2/rollercoaster.png)

Alright, let’s think about a forever loop in Lego Spike Prime like this:

Imagine you’re at an amusement park, and you find your favorite ride—the one you want to go on again and again! Instead of waiting in line every time, you get a special "forever" pass that lets you stay on the ride as long as you want. As soon as the ride finishes, it just starts again, no waiting necessary! That’s how a forever loop works in programming with Lego Spike Prime.

In a forever loop, the robot keeps doing the same thing—like spinning in a circle, flashing a light, or playing a sound—over and over again until you tell it to stop (or turn it off). It’s a super useful way to make your robot do something non-stop, just like you would if you had unlimited rides at an amusement park!

![jump](https://dp4rp4tcpuoo1.cloudfront.net/R400V2/floop.png)

The code example above will have the robot endlessly dancing in squares, keeping it moving until it either tires and the battery calls it quits!

Check it out: The forever loop has a flat bottom, like it's saying, "Nope, nothing's getting added here!" That’s because this loop just keeps going and going, looping forever without ever stopping to let anything else run.
